**Why Python?**

Python has emerged as one of the most popular general-purpose programming languages of the
twenty-first century. It was first released in 1991 and has steadily gained fans ever since. In this
course, you’ll use Python to learn and apply the fundamentals of programming – but why has it been
chosen over the many other languages available?

1. Python is versatile

Python has many different applications, including data science, games development, web development,
and machine learning. Whether you have ambitions to use programming to spot cyber security attacks,
build games, or code websites, understanding Python will give you a strong base from which to build
from.

2. Python is simple

Learning programming is not simple, but as languages go, Python is one of the easier languages
overall. Its syntax (the words, expressions and punctuation it uses) are similar to other popular
languages as well. Consider the following code, programmed in C (another programming language):

```py
fprintf(stdout,"Hello world!");
```

This statement in C would display “Hello world!” on your screen. In Python, the equivalent statement
is:

```py
sys.stdout.write("Hello world!")
```

Then in Java (yet another popular programming language), the equivalent statement would be:

```py
System.out.print("Hello world!");
```

Even at this early stage, you might see that there are similarities between the languages. Python’s
popularity over the other two languages can be attributed to how easy it is to get started.

3. Python is in demand

Python regularly features in the top three (if not the top) programming languages in demand – see
this article, for example. While there are some languages that are more suitable for education,
learning the fundamentals of programming in a popular language means you can easily transition from
study to industry.

4. Python has a great community of practice

While the supply of Python programmers does not meet demand, there is still a large community of
practice surrounding Python, offering many resources to help you learn and develop. While this
programming fundamentals course will set you up for success, when you complete the course you may
want to continue to develop your skills. Continuing to learn a well-supported, well-resourced
language with lots of guides, tips and advice online will allow you to make efficient use of your
time.

### 1.2.4 Valuing sequence

#### OVERVIEW - Running instructions in order

The code you’ve written so far contains one function and argument, but programs are typically made
up of many statements that are sequenced together to perform a task. In this task, you’ll learn
about the importance of sequencing, and apply this to your program.

Often, a program is compared to a recipe: just as a recipe needs the correct types and amounts of
ingredients, and the correct steps to be followed in a particular sequence, a program also needs its
data in correct types, and numbers and statements used in a specific sequence to achieve its goals.
In both, the sequence in which the steps or statements are performed affects the outcome: if we bake
a cake by putting the ingredients in the cake tin, baking them, then mixing them, it won’t work!

### 1.2.5 Working with error messages

#### OVERVIEW - Understanding problems in our programming

Just as a calculator would produce errors if you try to enter instructions or data in an incorrect
way, your program can also produce errors. In this task, you’ll learn to recognise three of the most
common types of errors.

Most programming errors fall in to one of three main categories:

1. Syntax errors
2. Run-time errors
3. Logic errors

Syntax errors

Syntax errors are errors that produce an error message as a result of incorrectly written
instructions. They are detected by the language editor, interpreter or compiler when executed. Try
removing the close quotation mark from the last print statement in hello_world.py and attempt to run
it. You will see an error as follows:

As you learnt earlier this week, like any language, Python has particular structural requirements
that must be met for it to be understood correctly. However, while incorrect English grammar in a
sentence or the misspelling of just one word would not make a report unreadable, a small syntax
error in a program would render the entire program unusable, untestable and arguably not a program.

Programs with syntax errors will result in zero marks if submitted for assessments in this course.

Therefore, especially at the start, new programmers must run and test their programs with each
statement they add to the program (hence, getting familiar with that F5 button!). A running program
with one line of code is better than one with a thousand but that has a syntax error.

#### Run-time errors

Some statements conform to the language’s grammar and syntax so are not (typically) detected by a
standard editor, interpreter or compiler but they perform an operation that should not be performed.
These represent run-time errors.

For example, if you ask the user to enter two numbers and write a program to divide the first number
by the second number, it will produce the correct result except if the second number is zero. This
is a limitation of the function. When dividing a number by zero, the interpreter will show an error
message (when the program is run, or at run-time) stating “ZeroDivisionError: division by zero”.

While the above results in a catastrophic failure of the program and prevents it from running
further, if the error only occurs when a particular combination of inputs is given to the program,
the program can remain somewhat usable and testable. While such an error would quickly become
apparently if given to thousands of users in a beta test, if it was submitted for an assessment,
such a program would still receive some marks for the parts that are testable. Penalties for
run-time errors will vary in your assessments throughout this course, so please check your
assessment specifications carefully.

This category refers to all other errors often attributed to mistakes made by the programmer. As you
will learn later this week, attention to detail is one of the key skills of programming!

For example, if we write a program intended to calculate 5 + 2 but instead calculate 5 – 2, there is
no syntax error but the logic is not what was needed (hence “logic error”).

Logic errors are often difficult to spot because you won’t necessarily be notified of them: you
might simply get an unexpected answer to a calculation, or worse, our program might just crash.

For the purpose of your assessments, logic errors only affect the part of the mark relevant to
demonstrating that code concept and functionality.

#### What's an algorithm? (4:57)

The video below explains what algorithms are, using an everyday example: Please watch the video Key
takeaways:

- Algorithms are a step-by-step set of instructions for solving a problem.
- Pseudo-code refers to code that represents human language but which resembles programming
  language.
- Well-structured algorithms can increase the efficiency of a program.
- “Buggy” algorithms are algorithms that contain a mistake.

In programming, an algorithm is a high-level statement of the structure of a program that is not
specific to a particular programming language. Look at the pseudo-code example below, which
represents an algorithm for outputting the sum of two values entered by the user:

```py
# Take two numerical inputs from user and store individually
# Calculate the sum of the two inputs
# Display the answer
```

Notice how even a trivial task such as calculating the sum of two values involves three steps! The
algorithm above is pseudo-code: it does not use any specific notation or programming language and
this is intentional. An algorithm captures only the details necessary for completing the task. This
might seem vague to a programmer, as it abstracts the programming-language-specific implementation
details, but it means we can share algorithms – as ways to approach problems – between different
languages.

#### Observe algorithms in practice

While algorithms tell us about the structure of a program, they don’t tell us how to implement it
within a language. Indeed, there can be many different implementations of an algorithm within one
language. To demonstrate this, try out the implementations of the algorithm above by copying and
pasting the code below into your Python editor as three separate programs.

Don’t worry that you don’t understand these functions yet – you’ll learn about them as we progress

When you run each program, you’ll be prompted to input “number 1” and “number 2”, before the program

#### Implementation 1

#### Implementation 2


As can be seen from the above examples, there can be multiple variations in the interpretation of an
algorithm, so long as each implementation meets the requirements of that algorithm.

While the programs you have worked with so far don’t contain a large number of functions, as your
programs develop, it will become increasingly important to trace through them, keeping track of the
operations and the changes to their values. In this task, you’ll learn how to use trace tables to
keep track of your programs' content.

Trace tables are a way of recoding the steps in a program, any expected inputs and outputs. Their
structure can vary depending on the preferences of the programmer, but here is an example using the
program you created earlier this week:

| Step | Statement                          | Notes            |
| ---- | ---------------------------------- | ---------------- |
| 1    | print("Hi world from the program") | Displays message |
| 2    | print("This is the second line")   | Displays message |

We’ll sometimes use trace tables (or a version of them) throughout this course to demonstrate the
logic behind the programs you create. When logic errors occur (i.e., ones that do not necessarily
show errors) creating trace tables can be an invaluable exercise.

The table itself can be created on a spreadsheet or simply on a piece of paper. It is also possible
to add statements to display the contents of variables, which can be convenient. However, doing so
clutters the program when you need it to be clutter-free the most; when you can’t figure out a logic
error.

### 1.4.3 Investigating error messages

#### OVERVIEW - Making sense of errors

While attention to detail can help you minimise the amount of errors in your programs, you will
inevitably encounter error messages as you develop your programming skills. In this task, you’ll
learn how to work with error messages and use them to fix your code.

To the end user who is not a computer-specialist, an error message on the screen could mean the end
of a productive day (or a month). To a programmer, however, an error message is all the help they
need to fix a problem (debug).

The syntax and run-time errors you learnt about earlier this week will almost always give you error
messages, while logic errors, due to their nature, will sometimes run through without causing an
error message. This is why logic errors are the hardest to spot.

#### Investigate the error message

Following this course closely will help you avoid errors, but eventually you’ll encounter them. You
should therefore develop your confidence in searching for and resolving error messages.

- Type the code below in to the editor.
- Run the program
- When prompted, enter any positive whole number for the first number, and **zero for the second
  number**.

```py
num1=int(input("Enter number 1: "))
num2=int(input("Enter number 2: "))
print(num1/num2)
```

You will see an output similar to the one below:

```py
Enter number 1: 3
Enter number 2: 0
Traceback (most recent call last):
File "hello_world.py", line 3, in <module> print(num1/num2)
ZeroDivisionError: division by zero
```

In the above example, the last line describes the error. Search for the error message text online,
and use the resources you encounter to work out what it means.

When searching for error message information, it’s good practice to exclude details of actual
outputs, files names, code and line numbers, as these will vary from program to program. The error
message itself, however, will almost always have been suffered by someone else who went to the great
community of Python programmers for help!

Getting familiar with error messages as you encounter them will help you avoid and respond to them
in the future.

### 1.4.4 Referring to the documentation

#### OVERVIEW - Resolving issues through the resources

Earlier this week you were introduced to the “textbook” for this course - the Python documentation.
In this task, you’ll learn about two extra Python resources that you can refer to throughout this

Programming languages are frequently updated with new features and functions. This course teaches
you the fundamentals of programming, but we also want you to develop the confidence to go on and
continue to learn Python or other languages beyond the course after completion. To enable this, it’s
important that you develop your skills in looking up and following the official documentation of a
language.

#### Read the documentation

The main page of Python’s official documentation can be accessed here , but the following sections
will be of particular use as you progress through this course and beyond:

- Language reference – describes the syntax of the Python language
- Library reference – describes the operations and components available in Python.

Read the introduction to both references to build your understanding of what they offer and how you
can use them.

Of course, the fundamentals of the language and the operations relevant to this course will be
taught to you as you progress through the material. However, these references provide additional
resources for you to refer to. As you develop your confidence in programming, they will also allow
you to start using new functions and other operations which will enhance the programs you create.
